Project Plan

Build an internal IT Service Management portal for a telecom Corporate IT department supporting incident ticketing, change request workflows, knowledge base, SLA tracking with alerts, and asset inventory linked to tickets. Users are end-users, IT technicians, and IT managers. Solution must use Django REST Framework, Vue 3 + TypeScript, PostgreSQL, and provide role-based dashboards and email notifications.

Total effort: 100hRecommended team: 4 peoplePhases: 2

Technology Stack

BackendDjango 5 + Django REST Framework on Python 3.11
FrontendVue 3 + TypeScript + Vite
DatabasePostgreSQL 15
InfrastructureDocker + docker-compose (dev) with readiness for deployment on AWS ECS Fargate
Key Librariesdjangorestframework, django-filter, django-rest-knox (or djangorestframework-simplejwt), psycopg2-binary, celery, redis, vue-router, pinia, axios, chart.js

Tasks

task_001 Design data model and implement core ITSM backend APIs MEDIUM 32h
Model incidents, change requests, assets, SLAs, and knowledge articles in Django. Implement DRF serializers, viewsets, and role-based permissions for end-user, technician, and manager, including ticket lifecycle (create, assign, escalate, resolve) and change approval endpoints.
Acceptance Criteria
  • Django models for incidents, change requests, assets, SLAs, and knowledge articles created and migrated to PostgreSQL
  • DRF endpoints for CRUD on incidents, change requests, assets, and knowledge articles available and documented via OpenAPI/Swagger
  • Role-based access control enforces end-user, technician, and manager permissions on all endpoints
  • Ticket state transitions (create, assign, escalate, resolve, close) validated and blocked when invalid
Deliverables
  • Django app with models, serializers, viewsets, and URL routing
  • Database migration scripts for PostgreSQL
  • OpenAPI/Swagger documentation for core ITSM endpoints
Phase: phase_1  |  Dependencies: None
task_002 Implement SLA tracking, breach alerts, and email notifications HIGH 16h
Add SLA policies, timers, and breach detection linked to incidents. Implement Celery-based background jobs and email notifications on ticket state changes and SLA breaches. Expose reporting endpoints for SLA compliance and MTTR by category.
Acceptance Criteria
  • SLA rules configurable per priority/category and stored in database
  • Background job marks SLA breaches and persists timestamps for response and resolution
  • Email notifications sent on create, assign, escalate, resolve, and SLA breach using test SMTP
  • API endpoints return SLA compliance rate and MTTR by category for a given date range
Deliverables
  • Celery tasks and Redis configuration for scheduled SLA checks
  • Django email backend configuration and notification templates
  • Reporting API endpoints for SLA metrics and MTTR
Phase: phase_1  |  Dependencies: task_001
task_003 Build Vue 3 frontend for incident, change, and asset management MEDIUM 24h
Create Vue 3 + TypeScript SPA with authentication, role-based navigation, and views for incidents, change requests, and assets. Support ticket creation, assignment, escalation, resolution, and linking assets to tickets for technicians and managers.
Acceptance Criteria
  • Login/logout flow integrated with backend auth and protected routes per role
  • End-users can create and view their incidents; technicians/managers can list, filter, and update all incidents
  • Change request creation, approval/rejection, and status updates available to authorized roles
  • Assets can be created, edited, and linked/unlinked to incidents from the UI
Deliverables
  • Vue 3 SPA with routed views for incidents, change requests, and assets
  • Pinia stores and Axios services for ITSM entities
  • Responsive layout compatible with latest Chrome and Edge
Phase: phase_2  |  Dependencies: task_001
task_004 Implement knowledge base and self-service portal UI LOW 12h
Develop knowledge base browsing and search for end-users, with article management for technicians/managers. Integrate article suggestions when creating incidents and provide a self-service landing page for common issues.
Acceptance Criteria
  • End-users can browse categories, search articles, and view article details
  • Technicians/managers can create, edit, publish/unpublish, and categorize articles
  • Incident creation form suggests relevant articles based on keywords/category
  • Self-service home page shows featured articles and quick links to raise tickets
Deliverables
  • Vue components for knowledge base list, detail, and search
  • Admin UI for article management with status controls
  • Integrated self-service landing page for end-users
Phase: phase_2  |  Dependencies: task_001, task_003

Create manager dashboard showing open tickets by priority, SLA compliance, and MTTR by category using Chart.js. Design and execute test cases for workflows, notifications, and role-based access; set up basic CI tests.
Acceptance Criteria
  • Dashboard displays open tickets by priority, SLA compliance %, and MTTR by category using backend metrics APIs
  • Automated API and UI smoke tests cover main flows for all three roles
  • Email notifications verified in test environment for key state changes and SLA breaches
  • Defects found during QA are logged and critical issues resolved before sign-off

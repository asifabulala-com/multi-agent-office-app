Project Plan

Develop a real-time NOC monitoring platform for telecom network engineers to ingest, correlate, and visualize events from 500+ devices. The system will provide incident management, live topology maps, alert escalation (PagerDuty/SMS), and SLA reporting, with strict real-time and reliability constraints.

Total effort: 80hRecommended team: 3 peoplePhases: 2

Technology Stack

BackendFastAPI + Python 3.11
FrontendReact 18 + TypeScript
DatabaseInfluxDB for time-series metrics, PostgreSQL 15 for incident management
InfrastructureDocker + AWS ECS
Key Librariesscikit-learn, SQLAlchemy, websockets, networkx, PagerDuty API client

Phases

Core Data Ingestion & Incident Engine
3 weeks
Deliver robust ingestion of SNMP, syslog, and NetFlow, and implement event correlation with incident management.
Tasks: task_001, task_002, task_003
Visualization, Escalation & SLA Reporting
2 weeks
Implement live topology dashboard, alert escalation, and SLA reporting module.
Tasks: task_004, task_005

Build FastAPI microservices to ingest SNMP traps, syslog, and NetFlow from 500+ devices, normalizing and storing data in InfluxDB.

  • Data schema documentation

Implement rule-based and ML-assisted event correlation, incident creation, and severity classification with PostgreSQL persistence.

  • Incidents persisted and updated in PostgreSQL
Deliverables
  • Correlation engine
  • Incident management schema
Phase: phase_1  |  Dependencies: task_001

Acceptance Criteria
  • P1/P2 incidents trigger PagerDuty and SMS notifications

Deliverables
  • Escalation module
  • API integration tests
Phase: phase_1  |  Dependencies: task_002
task_004 Live Topology Map and Device Health Dashboard MEDIUM 16h
Develop React dashboard with WebSockets for real-time topology visualization and device health status updates.

  • Topology map updates in real-time for device status changes
  • Device health indicators reflect backend data within 2 seconds
Deliverables
  • React dashboard
  • WebSocket backend
Phase: phase_2  |  Dependencies: task_001, task_002

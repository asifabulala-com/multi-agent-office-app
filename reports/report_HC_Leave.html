Project Plan

Build a web-based Leave & Attendance Management System for a telecom company’s Human Capital department. Employees, line managers, and HR admins will manage leave requests, attendance (with location tagging), and policy enforcement, with real-time leave balances and monthly reports. System must integrate with an existing employee directory REST API for org-aware approval workflows using FastAPI, React, and PostgreSQL.

Total effort: 110hRecommended team: 4 peoplePhases: 2

FrontendReact 18 + TypeScript
DatabasePostgreSQL 15
InfrastructureDocker + AWS ECS + AWS RDS (PostgreSQL) + AWS CloudWatch
Key LibrariesSQLAlchemy 2.x, Alembic, Pydantic, Authlib or PyJWT, Requests (for employee directory integration), React Router, Axios, Material UI (MUI), React Query, Jest + React Testing Library, Pytest

Build policy enforcement logic, attendance APIs with location tagging, and REST integration with the existing employee directory for org-aware approvals.
Acceptance Criteria
  • Policy engine enforces carry-over limits, blackout dates, and minimum notice on leave creation

  • Approval routing uses directory data to assign line manager approvers automatically
Deliverables

  • Attendance clock-in/out FastAPI endpoints

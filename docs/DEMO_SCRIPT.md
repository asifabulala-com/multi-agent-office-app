# Demo Video Script — Multi-Agent PM System
*Total target: 2 min 30 sec — 4 sections*

---

## Section 1 — Problem & Use Case  `0:00 – 0:30`
**Screen: main web UI at localhost:8000**

> "Software projects fail quietly.
> The planner makes decisions the engineer never sees.
> QA finds bugs the risk analyst should have predicted.
> The stakeholder learns about delays when it's too late to fix them.
>
> This system changes that.
> It connects five AI agents — a Project Manager, Engineer, QA, Risk Analyst, and Stakeholder —
> into one collaborative loop that mirrors how a real delivery team should work.
>
> It's built specifically for the enabling functions of a telecom company —
> HR, Finance, Corporate IT, Billing, Revenue Assurance —
> the teams that keep the business running behind the scenes."

---

## Section 2 — Agent Architecture  `0:30 – 1:00`
**Screen: architecture_diagram.html — point to each agent as named**

> "Here's the architecture.
> Five specialised agents, orchestrated by a LangGraph state machine.
>
> The Project Manager decomposes the brief into a phased plan with tech stack and acceptance criteria.
> The Risk Analyst flags delivery risks before any engineering starts.
> If risks are high enough, the PM replans automatically — that's a conditional edge in the graph.
> The Stakeholder approves the plan and can block it.
> The Engineer implements each task and writes real source files.
> The QA Agent critiques the output, raises specific issues, and drives a revision loop.
>
> Every handoff is logged, every decision recorded. Nothing is simulated."

## Recording checklist

- [ ] Browser window: Chrome, maximised, 1920×1080
- [ ] localhost:8000 open and server running (`python run.py`)
- [ ] Architecture diagram open in a separate tab (`docs/architecture_diagram.html`)
- [ ] CIT_008 project previously run — report at `/report/CIT_008` and MVP at `/mvp/CIT_008` ready as fallback
- [ ] Screen recorder: OBS or Windows Game Bar (`Win + G`)
- [ ] Microphone tested — or use the teleprompter TTS (`docs/demo_teleprompter.html`)
- [ ] Close notifications, set browser to full-screen
- [ ] Font size: browser zoom 110% for readability

## Tips for a tight recording
- Speak at 140 words/minute — slightly slower than normal conversation
- Pause 0.5s before each section change
- For the live run: if the simulation is slow, talk over it rather than waiting silently
- The teleprompter (`docs/demo_teleprompter.html`) can read the script aloud while you demonstrate — record both simultaneously
